# api — AI Inference API Server

`api/` 디렉토리는 본 프로젝트에서 **AI 추론 기능을 외부에 서비스 형태로 제공하는 API 서버**를 담당한다.

드론/CCTV 등으로부터 전달되는 이미지(또는 영상 프레임)에 대해  
AI 분석을 수행하고, 그 결과를 **표준화된 JSON 응답**으로 반환하는 것이 주된 역할이다.

---

## 📌 디렉토리 역할 요약

- AI 추론 요청을 **HTTP API** 형태로 수신
- 추론 결과를 **관제 시스템이 사용 가능한 구조화된 데이터**로 반환
- 실제 AI 모델 구현과 무관하게 **API 인터페이스를 안정적으로 유지**
- 이후 비동기 처리 구조로 확장 가능한 **서비스 진입점(API Gateway 역할)**

즉, `api/`는  
> **AI 모델을 감싸는 서비스 계층(Service Layer)** 이며,  
> 프로젝트 전체에서 “외부와 직접 통신하는 AI 서버” 역할을 수행한다.

---

## 🧱 설계 관점에서의 위치

프로젝트 전체 구조에서 `api/`는 다음과 같은 위치를 가진다.

- **입력**: 이미지/영상 분석 요청
- **처리**: AI 추론 (현재는 Stub, 이후 실제 모델)
- **출력**: 위험도, 우선순위, 객체 정보, 텍스트 설명

AI 모델 자체(YOLO, BLIP 등)는 교체 가능하지만,  
`api/`가 제공하는 **요청/응답 구조(API Contract)** 는 유지되는 것을 전제로 한다.

## 📁 디렉토리 구조

```text
api/
├─ app/
│  ├─ main.py        # FastAPI 엔트리포인트
│  ├─ schemas.py     # 요청/응답 데이터 모델 (API Contract)

├─ requirements.txt  # API 서버 의존성 목록
├─ run.sh            # API 서버 실행 스크립트
